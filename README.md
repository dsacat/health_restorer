# Health Restorer

`Health Restorer` — одноразовый сценарий глубокого обслуживания Windows 10/11. Он сам запрашивает права администратора, создаёт временное задание от имени `SYSTEM`, перезагружает компьютер между этапами и удаляет задание после завершения.

## Что выполняется

1. Обновление баз Microsoft Defender.
2. Автономная проверка Microsoft Defender до запуска обычной Windows.
3. Резервное копирование и отключение пользовательской автозагрузки:
   - значения `Run` и `RunOnce` для компьютера и локальных пользователей;
   - общая и пользовательские папки «Автозагрузка»;
   - сторонние задания Планировщика с запуском при входе или загрузке.
4. Предзагрузочная проверка и исправление системного диска.
5. Проверка всех локальных дисков.
6. Восстановление хранилища компонентов через DISM.
7. Проверка и восстановление системных файлов через SFC.
8. Очистка:
   - временных файлов Windows и пользователей;
   - загруженного кэша Windows Update;
   - Delivery Optimization;
   - отчётов об ошибках;
   - DNS-кэша;
   - корзины;
   - кэшей Chrome, Edge и Firefox;
   - кэша шейдеров и миниатюр.
9. Автоматическая оптимизация локальных дисков через `defrag /O`.
10. Создание журнала и итогового отчёта.

## Важное предупреждение

Перед запуском:

- сохрани все открытые файлы;
- подключи ноутбук к питанию;
- не выключай компьютер во время проверки диска, DISM или SFC;
- учитывай, что корзина и кэши браузеров будут очищены;
- учитывай, что отключённые программы перестанут запускаться автоматически.

Сценарий намеренно **не отключает драйверы, системные службы Windows и задания из ветки `\Microsoft\`**. Их полное отключение может привести к отсутствию сети, звука, входа в систему или загрузки Windows.

Сценарий не заменяет резервную копию данных и не гарантирует исправление физически неисправного накопителя.

## Запуск

Скачай репозиторий и запусти:

```text
Start Health Restorer.cmd
```

Подтверди запрос контроля учётных записей. Дальше вмешательство не требуется.

Компьютер перезагрузится минимум два раза:

1. для Microsoft Defender Offline;
2. для предзагрузочной проверки диска.

После последней загрузки обслуживание продолжится автоматически в фоне от имени `SYSTEM`.

## Где смотреть результат

Журнал:

```text
C:\ProgramData\HealthRestorer\health-restorer.log
```

Итоговый отчёт:

```text
C:\Users\Public\Desktop\HealthRestorer-report.txt
```

Резервная копия отключённой автозагрузки:

```text
C:\ProgramData\HealthRestorer\StartupBackup-ГГГГММДД-ЧЧММСС
```

## Восстановление автозагрузки

Открой PowerShell от имени администратора в папке проекта и выполни:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\HealthRestorer.ps1 -Mode RestoreStartup
```

Будет использована последняя резервная копия из:

```text
C:\ProgramData\HealthRestorer\latest-backup.txt
```

## Ограничения

- Для автономной проверки требуется активный Microsoft Defender. При отключённом Defender сценарий попытается выполнить обычную полную проверку.
- DISM может использовать Windows Update для загрузки исправных компонентов, поэтому желательно подключение к интернету.
- Проверка диска и полное обслуживание могут выполняться долго.
- Некоторые файлы кэша могут остаться, если Windows удерживает их открытыми.
- Настройки автозагрузки некоторых приложений могут восстановиться после их обновления или повторного запуска.
