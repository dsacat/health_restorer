# Health Restorer

`Health Restorer` — одноразовый сценарий глубокого обслуживания Windows 10/11. Он запрашивает права администратора, создаёт временные задания от имени `SYSTEM`, продолжает работу после перезагрузок и удаляет задания после завершения.

## Готовый EXE

```text
HealthRestorer.exe
```

Контрольная сумма:

```text
HealthRestorer.exe.sha256
```

EXE автономный и однофайловый. Отдельно устанавливать .NET и вручную размещать PowerShell-скрипты не требуется.

## Порядок работы

1. Обновляются базы Microsoft Defender.
2. Запускается Microsoft Defender Offline до обычной загрузки Windows.
3. После загрузки отключается безопасная сторонняя автозагрузка с резервной копией.
4. Выполняется предзагрузочная проверка системного диска.
5. DISM и SFC восстанавливают Windows.
6. Проверяются и оптимизируются доступные диски.
7. Очищаются временные файлы, кэши Windows и браузеров, DNS-кэш, эскизы и корзина.
8. Удаляются подтверждённые остатки не до конца удалённых программ.
9. Очищается доступное свободное пространство, в котором могут оставаться удалённые файлы: перезапись HDD и TRIM/ReTrim для SSD.
10. **Последним рабочим этапом** запускается Microsoft Defender Full Scan всех доступных файлов.
11. Программа не пишет «сканирование началось», пока Defender не подтвердит новое значение `FullScanStartTime`.
12. Завершение принимается только после подтверждённого нового `FullScanEndTime`; затем создаётся итоговый отчёт.

## Что означает «полная проверка всех файлов»

Microsoft Defender Full Scan запрашивается через:

```powershell
Start-MpScan -ScanType FullScan
```

Запрос выполняется в отдельном процессе. Программа сначала запоминает прежние `FullScanStartTime` и `FullScanEndTime`, затем ждёт появления нового времени начала. Только после этого в журнале появляется строка `CONFIRMED`. Завершение также принимается только после появления нового `FullScanEndTime`.

Проверяются все файлы, которые доступны Windows на смонтированных локальных томах и, при разрешённых настройках Defender, на подключённых съёмных и сетевых накопителях.

Невозможно проверить данные, физически недоступные операционной системе: отключённые диски, не загруженные локально облачные файлы, повреждённые нечитаемые области, заблокированные BitLocker-тома, некоторые зашифрованные контейнеры и архивы с неизвестным паролем.

Отчёт полной проверки:

```text
C:\ProgramData\HealthRestorer\defender-full-scan-report.txt
```

Сохранённые настройки Defender:

```text
C:\ProgramData\HealthRestorer\defender-preferences-before-full-scan.json
```

## Охват автозагрузки

Расширенный аудит сохраняется в:

```text
C:\ProgramData\HealthRestorer\autorun-audit.json
```

Он включает:

- стандартные и перенаправленные общие и пользовательские папки Startup;
- `Win32_StartupCommand`;
- машинные и пользовательские Run/RunOnce и Policies Run;
- Winlogon, AppInit DLL, AppCert DLL, IFEO, SilentProcessExit и Active Setup;
- ShellServiceObjectDelayLoad и ShellExecuteHooks;
- задания входа, загрузки, регистрации и смены пользовательской сессии;
- автоматические службы, загрузочные и системные драйверы;
- постоянные WMI-фильтры, потребители и привязки.

Абсолютно любой возможный способ закрепления программы перечислить невозможно: вредоносный код может использовать уязвимости, изменённые загрузчики, прошивки или новые техники. Поэтому программа сочетает Defender Offline, Full Scan и широкий аудит, но не обещает невозможную стопроцентную гарантию.

## Очистка ранее удалённых файлов

Сводка по каждому доступному тому сохраняется в:

```text
C:\ProgramData\HealthRestorer\deleted-data-summary.json
```

На HDD перезапись свободного пространства удаляет обычные остатки файлов, которые были удалены из корзины или удалены напрямую, но ещё оставались в неиспользуемых секторах файловой системы.

На SSD и флеш-памяти выполняется TRIM/ReTrim. Из-за контроллера, резервных ячеек и выравнивания износа обычная программа не может гарантировать физическую перезапись каждого NAND-блока. Для такой гарантии нужен Secure Erase/Sanitize всего накопителя, который уничтожает также Windows и все существующие файлы.

Нельзя очистить отключённые, несмонтированные, заблокированные, повреждённые, доступные только для чтения или нераспознанные тома.

## Защита при очистке программных остатков

Сценарий не удаляет произвольную папку только из-за неизвестного названия. Остаток удаляется лишь при сочетании нескольких признаков: отсутствует деинсталлятор, запись не относится к Windows Installer или Microsoft, а в каталоге отсутствуют работающие исполняемые компоненты.

Перед изменением сохраняются экспорт разделов реестра, XML заданий Планировщика, разделы реестра служб, копии битых ярлыков и JSON-отчёт.

Резервная копия метаданных:

```text
C:\ProgramData\HealthRestorer\ProgramResidueBackup-ГГГГММДД-ЧЧММСС
```

## Запуск

1. Сохрани открытые файлы.
2. Создай резервную копию важных данных.
3. Подключи компьютер к питанию.
4. Подключи все накопители, которые нужно проверить и очистить.
5. Запусти `HealthRestorer.exe`.
6. Подтверди запрос прав администратора.

Компьютер перезагрузится несколько раз. Полная проверка, проверка дисков и очистка свободного пространства могут продолжаться много часов.

## Ограничения

- Для антивирусных этапов требуется активный Microsoft Defender.
- Tamper Protection или корпоративная политика могут запретить изменение отдельных настроек и исключений; такие случаи фиксируются в журнале.
- DISM может использовать Windows Update.
- Открытые системой файлы кэша иногда невозможно удалить до следующей перезагрузки.
- Неизвестный тип накопителя не перезаписывается вслепую, чтобы не изнашивать возможный SSD; программа пытается выполнить ReTrim и записывает результат.
- Сценарий не заменяет резервное копирование и не исправляет физически неисправный накопитель.
