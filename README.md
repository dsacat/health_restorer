# Health Restorer

`Health Restorer` — одноразовый сценарий глубокого обслуживания Windows 10/11. Он сам запрашивает права администратора, создаёт временные задания от имени `SYSTEM`, перезагружает компьютер между этапами и удаляет задания после завершения.

## Скачать готовый EXE

Готовая 64-разрядная версия:

```text
dist/HealthRestorer.exe
```

Контрольная сумма:

```text
dist/HealthRestorer.exe.sha256
```

EXE является автономным однофайловым приложением: отдельная установка .NET и ручное размещение PowerShell-скриптов не требуются. При запуске он извлекает встроенные сценарии в `C:\ProgramData\HealthRestorer\Package` и запускает единый процесс обслуживания.

Файл пока не подписан платным сертификатом подписи кода. Windows SmartScreen может показать предупреждение о неизвестном издателе. Авторские данные в свойствах файла и цифровая криптографическая подпись — разные вещи.

## Что выполняется

1. Обновление баз Microsoft Defender.
2. Автономная проверка Microsoft Defender до запуска обычной Windows.
3. Резервное копирование и отключение пользовательской автозагрузки:
   - значения `Run` и `RunOnce` для компьютера и локальных пользователей;
   - общая и пользовательские папки «Автозагрузка»;
   - сторонние задания Планировщика с запуском при входе или загрузке.
4. Предзагрузочная проверка и исправление системного диска.
5. Проверка всех локальных дисков.
6. Восстановление хранилища компонентов через DISM.
7. Проверка и восстановление системных файлов через SFC.
8. Очистка временных файлов, кэшей Windows, браузеров, шейдеров, отчётов об ошибках, DNS-кэша и корзины.
9. Очистка подтверждённых остатков удалённых программ:
   - сломанных записей деинсталляции;
   - оставшихся каталогов, если деинсталлятор и значок отсутствуют, а в каталоге нет исполняемых компонентов;
   - сторонних заданий Планировщика, у которых отсутствуют запускаемые файлы;
   - сторонних служб, у которых отсутствует исполняемый файл;
   - битых ярлыков программ;
   - пустых каталогов программ.
10. Очистка следов уже удалённых файлов в свободном пространстве:
    - на HDD свободное место перезаписывается через `cipher /w`;
    - на SSD выполняется принудительный `TRIM/ReTrim`.
11. Автоматическая оптимизация локальных дисков через `defrag /O`.
12. Создание журнала и итогового отчёта.

## Как защищена очистка программных остатков

Сценарий не удаляет произвольные каталоги только потому, что их название неизвестно. Каталог считается остатком лишь при сочетании нескольких признаков: отсутствует деинсталлятор, отсутствует значок программы, запись не относится к Windows Installer или Microsoft, а в оставшемся каталоге нет исполняемых файлов и библиотек.

Перед изменением сохраняются:

- экспорт удаляемых разделов реестра;
- XML удаляемых заданий Планировщика;
- разделы реестра удаляемых служб;
- копии удаляемых битых ярлыков;
- итоговый JSON-отчёт.

Резервная копия метаданных:

```text
C:\ProgramData\HealthRestorer\ProgramResidueBackup-ГГГГММДД-ЧЧММСС
```

Удалённые каталоги программ в резервную копию не дублируются, иначе их данные продолжили бы занимать диск и не попали бы под последующую очистку свободного пространства.

## Важное предупреждение

Перед запуском:

- сохрани все открытые файлы;
- подключи ноутбук к питанию;
- создай резервную копию особо важных данных;
- не выключай компьютер во время проверки диска, DISM, SFC или перезаписи свободного пространства;
- учитывай, что корзина и кэши браузеров будут очищены;
- учитывай, что отключённые программы перестанут запускаться автоматически.

Сценарий намеренно **не отключает драйверы, системные службы Windows и задания из ветки `\Microsoft\`**. Их полное отключение может привести к отсутствию сети, звука, входа в систему или загрузки Windows.

Сценарий не заменяет резервную копию данных и не гарантирует исправление физически неисправного накопителя.

## Запуск EXE

Запусти:

```text
HealthRestorer.exe
```

Подтверди запрос контроля учётных записей. EXE содержит манифест `requireAdministrator`, поэтому Windows сразу запросит административные права. Дальше вмешательство не требуется.

Компьютер перезагрузится минимум два раза:

1. для Microsoft Defender Offline;
2. для предзагрузочной проверки диска.

После последней загрузки обслуживание продолжится автоматически в фоне от имени `SYSTEM`.

## Альтернативный запуск исходных сценариев

Для проверки или разработки можно скачать весь репозиторий и запустить:

```text
Start Health Restorer.cmd
```

## Файлы проекта

- `dist/HealthRestorer.exe` — готовый автономный Windows x64 EXE.
- `dist/HealthRestorer.exe.sha256` — SHA-256 готового EXE.
- `src/HealthRestorer.Launcher/` — исходный код однофайлового .NET-запускателя.
- `Start Health Restorer.cmd` — точка запуска исходных PowerShell-сценариев.
- `Start-HealthRestorer.ps1` — устанавливает одноразовый завершающий этап и запускает основной сценарий.
- `HealthRestorer.ps1` — проверка Defender, дисков, Windows, очистка мусора и оптимизация.
- `ProgramResidueCleanup.ps1` — удаление подтверждённых остатков деинсталлированных программ.
- `SecureDeletedData.ps1` — запускает очистку программных остатков, затем очищает свободное пространство и удаляет собственное задание.
- `.github/workflows/build-exe.yml` — автоматическая сборка, синтаксическая проверка PowerShell и проверка метаданных EXE на Windows runner.

```powershell

```

## Ограничения

- Для автономной проверки требуется активный Microsoft Defender. При отключённом Defender сценарий попытается выполнить обычную полную проверку.
- DISM может использовать Windows Update для загрузки исправных компонентов, поэтому желательно подключение к интернету.
- Проверка диска и полное обслуживание могут выполняться долго. Перезапись свободного места большого HDD может занять несколько часов.
- Очистка остатков программ использует консервативные признаки и намеренно пропускает сомнительные каталоги, MSI-пакеты, компоненты Microsoft и объекты с существующими исполняемыми файлами.
- На HDD `cipher /w` перезаписывает свободное пространство, в котором могут находиться остатки удалённых файлов.
- На SSD используется `TRIM/ReTrim`: из-за контроллера, резервных ячеек и выравнивания износа обычная программа не может гарантировать физическую перезапись каждого блока. Для гарантированного уничтожения всех данных SSD требуется Secure Erase/Sanitize всего накопителя, что также удалит Windows и все оставшиеся файлы.
- Если тип накопителя определить не удалось, выполняется только `ReTrim`, чтобы случайно не изнашивать SSD многократной записью.
- Некоторые файлы кэша могут остаться, если Windows удерживает их открытыми.
- Настройки автозагрузки некоторых приложений могут восстановиться после их обновления или повторного запуска.
